Here are some of the basic commands for git

1. git clone <url>
creates a copy of the repo
2. git fork <url>
creates a copy of someone else's repo
3. git add <filename> or git add * 
adds file to staging area. git will include the changes in the next revision to the repo.
4. git commit or git commit -m "commit message"
make a commit duh
5. git diff
shows all the changes made that has yet not been committed
6. git commit -am "message"
adds and then commits in one go
7. git status
tells status of the whole repo
8. git push or git push origin master
sends commit changes to remote repository (in master branch in the second case)
9. git pull
receives recent changes from remote repo

Merge Conflicts:

manually resolve and commit again

10. git log
shows a log of all commits and messages
11. git reset --hard <commit hash> or git commit --hard origin/master
reverts code back to the specified version

Branching:
12. git branch
shows all branches
13. git branch <new_branch_name>
create a new branch
14. git checkout <branch_name>
switch to the branch
15. git checkout -b <new_branch_name>
create a branch and then switch to it 
16. git merge <branch_name>
merges that branch to current branch
